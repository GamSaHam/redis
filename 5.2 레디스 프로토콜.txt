5.2 레디스 프로토콜
레디스 서버가 수신하는 모든 명령은 레디스 프로토콜을 통해서 송수신 된다.
데이터 통신을 위한 채널은 TCP/IP이며, 레디스 서버 환경파일에 기술된 포트를 사용한다.
레디스 프로토콜은 마치 HTTP 프로토콜과 같은 텍스트 기반 프로토콜이며 사용하기 쉽고 사람이 읽기도 쉽다.

프로토콜 구조
레디스 프로토콜은 바이너리 세이프(binary-safe) 하다. 전송되는 데이터에 따라서 프로토콜이 오작동 하지 않는다.

어떤 프로토콜이 데이터의 전송 종료를 알리기 위해 줄바꿈(CR-캐리지리턴,LF-라인피드) 문자를 사용한다고 가정하자.
이때 텍스트 파일을 데이터로 전송하면 데이터가 전부 전송되지 못하고 종료되는 현상이 벌어진다.
이런 현상이 발생하는 이유는 텍스트 파일에 포함된 줄바꿈 문자가 프로토콜의 종료 구분자로 인식되어
데이터를 모두 전송하지 못하고 프로토콜이 종료되기 때문이다.(이러한 현상이 발생하는 프로토콜들을 
바이너리 세이프하지 않은 프로토콜이라 한다.)

*<인자의 개수> CRLF // 전송할 인자의 개수를 표시하는 구문
$<첫 번째 인자의 바이트 수> CRLF // 입력되는 구문
<인자> CRLF // 입력되는 구문
$<두 번째 인자의 바이트 수> CRLF
<인자> CRLF
...
$<N 번째 인자의 바이트수> CRLF
<인자> CRLF

set mykey value1 
첫번째인자 3바이트
두번째인자 5바이트
세번째인자 6바이트

*3
$3
set
$5
mykey
$6
value1

프로토콜 구조는 매우 단순하다.

telnet 을 사용 설정을 하고 putty 를 통해 접속을 해보자
*3
$3
set
$4
key1
$7
myvalue
+OK

응답기호는 +이다
에러 표시는 -이다
-1은 (nil) 을 나타낸다.

입력하면 레디스 항목에서 키가 저장되는것을 확인할 수 있다.
