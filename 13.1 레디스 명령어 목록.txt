
레디스 버전은 2.6.13이며, 이버전에서 제공하는 명령어 개수는 총 146개 이다.

명령은 문자열, 해시, 리스트, 셋, 정렬된 셋, 발행/구독, 트랜잭션, 스크립팅, 커넥션, 서버 관리 명령어 
순으로 나열

B.1 키 관리 명령어

del 키[키]
시간복잡도: O(1)
설명: 주어진 키를 삭제한다.
응답: <숫자응답> 삭제된 키의 수

dump 키
시간복잡도: O(N)
설명: 주어진 키의 값을 직렬화 하여 돌려준다. 직렬화된 값은 restore 명령으로 역 직렬화 할 수 있다.
응답: <벌크응답> 직렬화된 데이터, 키가 존재하지 않으면 nil

exists 키
시간복잡도: O(1)
설명: 주어진 키가 존재하는지 확인한다.
응답: <숫자응답> 존재하면 1, 존재하지 않으면 0

expire 키 만료시간
시간복잡도: O(1)
설명: 주어진 키의 만료시간을 초 단위로 설정한다. 지정된 만료시간이 지나면 키가 삭제된다.
응답: <숫자응답> 설정되었으면 1, 설정된 키가 없으면 0

expireat 키 타임스탬프
시간복잡도: O(1)
명령 설명: 주어진 키의 만료 일자를 유닉스 타임스탬프값으로 설정한다.
응답: <숫자응답> 설정되었으면 1, 설정된 키가 없으면 0

keys 패턴
시간 복잡도: O(N)
명령 설명: 주어진 glob-style 패턴에 매칭되는 키 목록을 조회한다.
응답: <멀티 벌크응답> 조회된 키 목록

migrate host port 키 '대상 데이터베이스' timeout
시간 복잡도: O(N)
명령 설명: 명령이 실행되는 레디스 서버에 저장된 키를 인자에 주어진 대상 레디스 서버로 복사한다.
응답: <상태응답> 성공은 OK, 실패는 ERR

move 키 '데이터베이스 인덱스'
시간 복잡도: O(1)
명령 설명: 주어진 키를 지정한 데이터베이스로 복사한다.
응답: <숫자응답> 복사 성공 1, 복사 실패 0

object 보조명령어 [인자[인자 ...]]
시간 복잡도: O(1)
명령 설명: object 명령은 객체의 상태를 조사할 수 있는 보조 명령어를 지원한다. 
- refcount 는 키가 참조된 횟수 정보를 돌려준다.
- encoding 는 레디스에 저장된 객체의 인코딩 상태 정보를 돌려준다.
- idletime 는 주어진 키에 대하여 마지막 읽기나 쓰기 요청이 일어난 이후의 시간을 조회 한다.

사용된 보조 명령에 따라서 응답의 종류가 변경된다.

응답: <숫자응답 또는 벌크응답> 조사 결과

persist 키
시간 복잡도: O(1)
명령 설명: 주어진 키의 만료시간을 해제한다.
응답: <숫자응답> 해제되었으면 1, 키가 존재하지 않으면 0

pexpire 키 만료시간
시간 복잡도: O(1)
명령 설명: 주어진 키의 만료시간을 밀리초 단위로 설정한다. expire 명령과 동일한 기능을 지원한다.
응답: <숫자응답> 설정되었으면 1, 키가 존재하지 않으면 0

pexpireat 키 '밀리초로 환산된 타임스탬프'
시간 복잡도: O(1)
명령 설명: 주어진 키의 만료 일자를 UNIX 타임스탬프 * 1000의 값으로 설정한다.
응답: <숫자응답> 설정되었으면 1, 키가 존재하지 않으면 0

pttl 키
시간 복잡도: O(1)
명령 설명: 주어진 키의 남은 만료시간을 밀리초 단위로 조회한다.
응답: <숫자응답> 밀리초 단위의 남은 만료시간

randomkey
시간 복잡도: O(1)
명령 설명: 인스턴스에 저장된 키 중에서 임의의 키를 조회한다.
응답: <벌크응답> 조회된 키

rename "변경할 키" "변경될 키"
시간 복잡도: O(1)
명령 설명: 주어진 키의 이름을 변경한다. 변경될 키가 이미 존재하면 값을 덮어쓴다.
응답: <상태응답> 성공하면 OK, 실패하면 ERR

sort 키 [BY 패턴][LIMIT '오프셋 건수'][GET 패턴 [GET 패턴...]]
[ASC|DESC][ALPHA][STORE '저장될 키']
시간 복잡도: O(N)
명령 설명: 주어진 키에 저장된 값으 ㄹ정렬한다. 단, 리스트, 셋, 정렬된 셋 데이터에 대해서만 처리 가능하다.
패턴에 지정되는 값은 glob-style 패턴 표현식이다.
응답: <멀티 벌크응답> 정렬된 목록

ttl 키
시간 복잡도: O(1)
명령 설명: 주어진 키의 만료시간은 초 단위로 조회한다.
응답: <숫자응답> 만료시간이 설정되지 않았으면 -1, 아니면 남은 만료시간

type 키
시간 복잡도: O(1)
명령 설명: 주어진 키에 저장된 자료의 형을 조회한다.
응답: <벌크응답> 조회된 자료의 형, 단, 키가 존재하지 않으면 none이라는 벌크응답을 돌려준다.

B.2 문자열 데이터 명령

append 키 값
시간복잡도: O(1)
명령 설명: 키가 존재하면 주어진 값을 저장된 값의 뒤에 추가한다. 단, string 타입 값에만 해당한다.
키가 존재하지 않으면 set 명령어와 동일하게 동작한다.
응답: <숫자응답> 추가된 문자열을 포함한 전체 문자열의 길이

bigcount 키 [시작인덱스][종료인덱스]
시간복잡도: O(N)
명령 설명: 키 값의 시작위치부터 종료위치까지의 바이트 범위에서 bit의 값이 1인 개수를 조회한다.
응답: <숫자응답> 지정된 범위에 저장된 비트 중 1의 개수

bitop 비트연산자 대상키 원본키1 [원본키2]
시간복잡도: O(N)
명령 설명: 입력되는 키에 저장된 비트값에 대한 비트연산을 결과를 대상키에 저장한다. 지정 가능한 비트 연산자
는 AND, OR, XOR, NOT 이다.
응답: <숫자응답> 대상키의 바이트 길이

decr 키
시간 복잡도: O(1)
명령 설명: 키에 저장된 값 을 1감소시킨다. 값이 숫자일 때만 해당된다.
응답: <숫자응답> 명령이 실행된 후의 키의 값

decrby 키 값
시간복잡도: O(1)
명령 설명: 키에 저장된 값을 주어진 값 만큼 감소시킨다.
응답: <숫자응답> 명령이 실행된 후의 키의 값

get 키
시간복잡도: O(1)
명령 설명: 키에 저장된 값을 조회한다.
응답: <벌크응답> 조회된 키의 값

getbit 키 오프셋
시간복잡도: O(1)
명령 설명: 키에 저장된 값에서 오프셋 위치에 해당하는 bit의 값을 조회한다.
응답: <숫자응답> 해당 오프셋 위치는 bit 값, 0 또는 1

getrange 키 시작위치 종료위치
시간복잡도: O(N)
명령 설명: 키의 문자열값에서 시작위치와 종료위치의 범위에 해당하는 문자를 조회한다. 프로그래밍 언어의
substring과 동일한 기능을 수행한다.
응답: <벌크응답> 조회된 문자열

getset 키 값
시간복잡도: O(1)
명령 설명: 주어진 키에 입력된 값을 저장하고 이전에 저장된 값을 돌려준다. 주어진 키가 존재하지 않으면
nil을 돌려준다.
응답: <벌크응답> 이전에 저장된 값

incr 키
시간복잡도: O(1)
명령 설명: 키 값을 1 증가시킨다. 단, 값과 이전 값이 숫자인 때만 해당된다.
응답: <숫자응답> 명령이 실행된 후의 키의 값

incrby 키 값
시간 복잡도: O(1)
명령 설명: 키의 값을 주어진 값만큼 감소시킨다. 단, 지정된 값과 저장된 값이 숫자인 때만 해당한다.
응답: <숫자응답> 명령이 실행된 후의 키의 값

incrbyfloat 키 증감값
시간 복잡도: O(1)
명령 설명: 키의 값을 주어진 증감값만큼 증가시킨다. 단, 지정된 값과 저장된 값이 배정밀도 float 숫자일때에만
 실행한다.
응답: <벌크응답> 명령어 실행된 후의 키의 값

mget 키[키 ...]
시간복잡도: O(1)
명령 설명: 입력된 모든 키의 값을 조회한다.
응답: <멀티 벌크응답> 조회된 키의 값 목록

mset 키 값[키 값...]
시간복잡도: O(1)
명령 설명: 입력된 모든 키와 값을 저장한다.
응답: <상태응답> 항상 OK

msetnx 키 값[키 값 ...]
시간 복잡도: O(1)
명령 설명: 주어진 키가 존재하지 않으면 모든 키와 값을 저장한다. 단, 입력된 키가 하나라도 이미 존재하면 모든 키가 저장
되지 않는다.
응답: <숫자응답> 주어진 모든 키가 저장되었을 때 1, 저장되지 않았을 때 0

psetex 키 '밀리초 값'
시간 복잡도: O(1)
명령 설명: key와 값을 입력하고 만료시간을 밀리초 단위로 설정한다.
응답: <상태응답> 저장되었으면 OK

set 키 값
시간 복잡도: O(1)
명령 설명: 입력된 키와 값을 저장한다.
응답: <상태응답> 항상 OK

setbit 키 오프셋 값
시간 복잡도: O(1)
명령 설명: 오프셋 위치에 bit값을 설정한다.
응답: <숫자응답> 이전에 저장되었던 값

setex 키 seconds 값
시간 복잡도: O(1)
명령 설명: 키에 값을 입력하고 만료시간을 초 단위로 설정한다. set 명령과 expire 명령을 합친 명령이다.
응답: <상태응답> 성공일 때 OK

setnx 키 값
시간 복잡도: O(1)
명령 설명: 키가 존재하지 않을 떄 값을 입력한다. 키가 존재할 떄는 값을 입력하지 않는다.
응답: <숫자응답> 저장되었으면 1, 저장에 실패하면 0

setrange 키 오프셋 값
시간 복잡도: O(N)
명령 설명: 문자열의 오프셋 위치에 입력된 값을 덮어쓴다. 만약 지정된 오프셋 위치가 존재하지 않으면 문자열이
오프셋 길이만큼 길어진다.
응답: <숫자응답> 변경된 이후의 전체 문자열 길이

strlen 키
시간 복잡도: O(1)
명령 설명: 지정된 문자열의 길이를 조회한다.
응답: <숫자응답> 저장된 문자열의 길이

B.3 해시 데이터 명령

hdel 키 필드[필드 ...]
시간 복잡도: O(1)
명령 설명: 주어진 키에 저장된 필드를 삭제한다. 입력으로 주어진 모든 필드를 삭제한다.
응답: <숫자 응답> 삭제된 필드의 수

hexists 키 필드
시간 복잡도: O(1)
명령 설명: 주어진 키에 저장된 필드의 존재 여부를 조회한다.
응답: <숫자응답> 주어진 필드가 존재한다면 1, 아니면 0

hget 키 필드이름
시간 복잡도: O(1)
명령 설명: 주어진 키에 저장된 필드의 존재 여부를 조회한다.
응답: <숫자응답> 주어진 필드가 존재하면 1 , 아니면 0

hgetall 키
시간 복잡도: O(N)
명령 설명: 주어진 키에 저장된 모든 필드와 값을 조회한다.
응답: <멀티 벌크응답> 주어진 해시의 모든 필드와 값의 목록

hincrby 키 필드 지정값
시간 복잡도: O(1)
명령 설명: 주어진 키에 저장된 필드의 값을 지정값 만큼 증가시킨다. 값에는 음수도 지정 가능하다.
응답: <숫자응답> 명령이 수행되고 난 후의 필드 값

hincrbyfloat 키 필드 지정 값
시간 복잡도: O(1)
명령 설명: 주어진 키에 저장된 필드 값을 해당 float값 만큼 증가시킨다. 단, 값이 배정밀도 float값이어야 한다.
최대 소수점은 17자리로 고정되어 있다.
응답: <벌크응답> 명령이 수행되고 난 후의 필드 값

hkeys 키
사간 복잡도: O(1)
명령 설명: 주어진 키에 저장된 모든 필드 목록을 조회한다.
응답: <멀티 벌크응답> 주어진 해시의 모든 필드 목록

hlen 키
시간 복잡도: O(1)
명령 설명: 주어진 키에 저장된 필드의 수를 조회한다.
응답: <숫자응답> 주어진 해시에 저장된 필드 개수

hmget 키 필드[필드 ...]
시간 복잡도: O(1)
명령 설명: 주어진 키에 저장된 필드 중에서 입력된 필드 값을 조회한다.
응답:<멀티 벌크 응답> 주어진 필드의 값

hmset 키 필드 값[필드 값] ...
시간 복잡도: O(1)
명령 설명: 입력된 모든 필드와 값을 주어진 키에 저장한다. 단, 주어진 키가 존재하지 않으면 새로운 키를 생성한다.
응답: <상태응답> 항상 OK

hset 키 필드 값
시간 복잡도: O(1)
명령 설명: 입력된 필드와 값을 주어진 키에 저장한다. 단, 입력된 필드가 이미 존재하면 기존 값이 업데이트 된다.
응딥: <숫자응답> 존재하지 않은 필드일 때 1, 존재하는 필드일 때 0

hsetnx 키 필드 값
시간 복잡도: O(1)
명령 설명: 요청한 필드와 값을 주어진 키에 저장한다. 단, 요청한 필드가 존재하지 않을 때만 처리된다.
응답: <숫자응답> 값이 저장되면 1, 아니면 0

hvals 키
시간 복잡도: O(N)
명령 설명: 주어진 키에 저장된 모든 값 목록을 조회한다.
응답: <멀티 벌크응답> 주어진 키에 저장된 모든 값 목록

B.4 리스트 데이터 명령
리스트 데이터 하나에 총 2^32-1개의 요소를 저장할수 있다.

blpop 키[키 ...] 대기시간
시간 복잡도: O(1)
명령 실행: 주어진 키들에 저장된 요소 중에서 왼쪽 끝의 요소를 조회하고 해당 요소를 제거한다.
단, 대기시간이 0일 떄는 요소가 입력될 때까지 대기한다. 주어진 키들에 모두 요소가 존재할 때는 명령에 입력
된 첫 번째 키에서 요소를 조회한다. 대기시간은 초 단위이며 대기시간 동안 데이터가 입력되지 않으면 nil을
반환한다.
응답: <멀티 벌크응답> 조회된 요소

brpop 키[키 ...] 대기시간
시간 복잡도: O(1)
명령 설명: 주어진 키들에 저장된 요소 중에서 오른쪽 끝의 요소를 조회하고 해당 요소를 제거한다. 단, 대기 시간이
0일 떄는 요소가 입력될 때까지 대기한다. 
응답: <멀티 벌크응답> 조회된 요소

brpoplpush 워놉ㄴ키 대상키 대기시간
시간 복잡도: O(1)
명령 설명: 원본키의 오른쪽 끝 요소를 제거하여 대상키의 왼쪽 끝에 저장한다. 단, 대기시간이 0일 떄는 원본키에
요소가 입력될 때까지 대기한다.
응답: <멀티 벌크응답> 원본키 리스트에서 조회된 값

lindex 키 조회인덱스
시간 복잡도: O(N)
명령 설명: 주어진 키의 인덱스 위치에 해당하는 요소를 조회한다. 단, 인덱스에 음수가 지정되면 리스트의
오른쪽 끝부터 역순 인덱스가 적용된다.
응답: <벌크응답> 해당 위치에 저장된 요소

insert 키 BEFOTRE|AFTER '피벗 대상요소' 요소
시간 복잡도: O(N)
명령 설명: 주어진 키에 저장된 요소 중에서 피벗 대상값과 일치하는 위치의 앞 또는 뒤에 요소를 추가한다.
응답: <숫자응답> 명령이 성공하면 리스트의 요소 수, pivot값을 찾지 못했을 때 -1

ilen 키
시간 복잡도: O(1)
명령 설명: 주어진 키에 저장된 요소의 수를 조회한다.
응답: <숫자응답> 주어진 리스트의 요소 수

lpop 키
시간 복잡도: O(1)
명령 설명: 주어진 키의 맨 왼쪽 끝 요소를 제거하고 돌려준다.
응답: <벌크응답> 조회된 요소값

lpush 키 요소[요소 ...]
시간 복잡도: O(1)
명령 설명: 주어진 키의 왼쪽 끝에 요소들을 입력한다. 인자로 여러 개의 요소가 입력되면 입력된 순서대로 저장된다.
응답: <숫자응답> 명령이 수행되고 난 후의 요소 수

lpushx 키 요소
시간 복잡도: O(1)
명령 설명: 주어진 키의 왼쪽 끝에 입력된 요소를 저장한다. 단, 주어진 키가 존재하지 않을 때는 입력하지 않는다.
응답: <숫자응답> 성공이면 명령이 수행되고 난 후의 요소 수, 실패하면 0

lrange 키 시작인덱스 종료인덱스
시간 복잡도: O(N)
명령 설명: 주어진 키의 시작인덱수부터 종료인덱스 범위의 요소를 조회한다. 단 지정된 키가 존재하지 않으면 nil을 돌려준다.
응답: <멀티 벌크응답> 해당 범위의 요소들, 존재하지 않을 때 nil

lrem 키 개수 요소
시간 복잡도: O(N)
명령 설명: 주어진 키의 요소 중 입력된 요소와 동일한 값을 가진 요소를 입력된 개수만큼 제거한다. 단 인덱스에
응수가 지정되면 리스트의 마지막 데이터부터 역순 인덱스가 적용된다.
응답: <숫자응답> 제거된 요소의 수

lset 키 인덱스 요소
시간 복잡도: O(N)
명령 설명: 주어진 키의 인덱스 위치에 해당하는 요소의 값을 변경한다. 단, 인덱스에 음수가 지정되면 리스트의 오른
쪽 끝부터 역순 인덱스가 적용된다. 단, 주어진 키가 존재하지 않으면 오류를 발생한다.
응답: <상태응답> 성공 OK, 해당 index가 없을 때 ERR

ltrim 키 시작인덱스 종료인덱스
시간 복잡도: O(N)
명령 설명: 주어진 키의 시작인덱스부터 종료인덱스 범위의 요소만 남기고 삭제한다. 단, 종료인덱스가 시작인덱스부
보다 작을 때는 모든 요소가 삭제도니다.
응답: <삭태응답> 항상 OK

rpop 키
시간 복잡도; O(1)
명령 설명: 주어진 키의 오른쪽 끝에 저장된 요소를 제거하고 그 값을 조회한다.
응답: <벌크응답> 조회된 요소값

rpoplpush 원본키 대상키
시간 복잡도: O(1)
명령 설명: 원본키의 오른쪽 끝에 저장된 요소를 제거하여 대상키의 왼쪽 끝에 저장한다.
응답: <벌크응답> 원본키 리스트에서 조회된 값

rpush 키 요소[요소 ...]
시간 복잡도: O(1)
명령 설명: 주어진 키의 오른쪽 끝에 요소를 추가한다. 인자로 여러 개의 요소가 입력되면 입력된 순서대로 저장된다.
응답: <숫자응답> 명령이 수행되고 난 후의 요소 수

rpushx 키 요소
시간 복잡도: O(1)
명령 설명: 주어진 키의 오른쪽 끝에 요소를 추가한다. 단, 주어진 리스트가 존재하지 않을 때는 생성하지 않는다.
응답: <숫자응답> 명령이 수행되고 난 후의 요소 수
